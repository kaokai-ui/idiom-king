# 我是成語王 程式 Review（2026-06-11）

## Findings

### 1. 關卡生成失敗時會默默退回前一關

- 檔案：`src/game/useChainState.ts:209-216`
- 嚴重度：高

目前 `loadLevel(lvl)` 在 `generateLevel()` 回傳 `null` 時，如果 `lvl > 1`，會直接呼叫 `loadLevel(lvl - 1)`。

影響：

- 玩家點「下一關」時，可能實際被送回前一關。
- 關卡編號看起來會倒退，造成進度混亂。
- 如果某些高關卡較難生成，玩家會反覆玩到前面的關卡而不自知。

建議：

- 不要自動退回前一關。
- 改成保留目標關卡編號，顯示「此關生成失敗，請重試」或自動重試同一關固定次數。

### 2. 棋盤尺寸不會跟著視窗 resize / 旋轉即時重算

- 檔案：`src/components/ChainBoard.tsx:29-37`
- 嚴重度：中

`cellSize` / `fontSize` 的 `useMemo` 只依賴 `[cols, rows]`，但實際計算又讀了 `window.innerWidth` 與 `window.innerHeight`。

影響：

- iPad 橫直切換後，若 rows / cols 不變，棋盤尺寸可能沿用舊值。
- 桌機調整視窗大小、分割視窗、瀏覽器 zoom 變動後，棋盤可能不會立即更新。
- 這類狀況很容易再次引發版面擠壓或錯位問題。

建議：

- 抽成 `useViewportSize()` 或 `ResizeObserver` 驅動的 state。
- 將 viewport width / height 納入重新計算依賴。

### 3. 測試批次快取沒有版本化內容校驗，演算法更新後可能讀到舊資料

- 檔案：`src/components/IdiomChainBatchTestScreen.tsx:16-17`
- 檔案：`src/components/IdiomChainBatchTestScreen.tsx:75-82`
- 嚴重度：中

測試頁目前只用固定 `batchId` 與 `seedBase` 驗證 localStorage 內容。只要這兩個值沒改，舊批次就會被直接重用。

影響：

- 如果之後 `levelGenerator` 改了演算法，但忘記 bump `batchId`，測試頁會繼續顯示舊棋盤。
- 這會讓人工測試結果和實際最新程式不一致，增加 debug 誤判風險。

建議：

- 在快取資料中加入 `generatorVersion`。
- 或直接加入 layout checksum / schema version，演算法一變就自動失效重建。

## Refactor 建議

### 1. 把棋盤尺寸常數集中到共用模組

- 參考檔案：`src/components/ChainBoard.tsx:11-15`
- 參考檔案：`src/game/levelGenerator.ts:224-231`

目前 `MIN_CELL`、`GAP`、`BOARD_PAD`、`BOARD_BORDER`、`GAME_PAD` 同時存在於 render 端與 generator 端。

風險：

- 任何一邊改數值，另一邊忘記同步，就可能再出現「生成器判定可放，但畫面實際塞不下」的問題。

建議：

- 建立例如 `src/game/boardLayout.ts`
- 將常數與 `calcCellSize` / `canFitBoardInViewport` 集中管理

### 2. 把 50 關測試頁拆成「批次資料層」與「顯示層」

- 參考檔案：`src/components/IdiomChainBatchTestScreen.tsx:92-175`

目前同一個元件同時負責：

- 批次生成
- localStorage 讀寫
- 複製 JSON
- 錯誤處理
- 畫面渲染

建議：

- 拆成 `useChainTestBatch()` hook
- UI 元件只接收 `batch/currentRecord/actions`

這樣後續如果要加：

- 問題關卡標記
- 匯出所有測試資料
- 指定關卡跳轉

會比較好維護。

### 3. 為關卡生成器補更明確的可重現測試

- 參考檔案：`src/game/levelGenerator.ts:133-145`

目前已經支援 seed random，也有基本的 deterministic 測試，但還可以再補：

- 同一 seed 產生相同 `presetCells`
- 同一 seed 產生相同 `charBank`
- 生成結果一定滿足「所有 active cell 坐標唯一且不重疊」

這能降低未來修改 generator 時，把舊問題帶回來的風險。

## 建議優先順序

1. 先修 `loadLevel` 自動退回前一關
2. 再補 `ChainBoard` 的 viewport resize / orientation 重新計算
3. 最後整理測試頁快取版本與生成器共用常數
